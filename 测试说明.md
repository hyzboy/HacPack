# main.cpp 测试说明

本文档用于说明 `main.cpp`（HacPack CLI）当前实现的测试方法与验收标准。

## 1. 测试目标

覆盖以下命令行为：

- `pack <directory> <output.hac>`
- `unpack <input.hac> <output_directory>`
- `list <input.hac> [--hash]`
- `compare <input.hac> <directory>`

并验证以下关键点：

- 仅支持 V2 包（`version=2`）
- `file_id=0` 为插件路由（插件名 -> file_id）
- `NameList` 插件可恢复文件名
- `CRC32` 插件可用于 `list --hash` 与 `compare`
- 未知插件名不会导致崩溃（可跳过）

---

## 2. 环境准备

在项目根目录执行：

```powershell
cmake -S . -B build -DBUILD_TESTING=ON
cmake --build build --config Debug
```

可执行文件路径：

- `build/bin/HacPack.exe`

---

## 3. 手工测试步骤

以下示例假设当前目录为项目根目录。

### 3.1 打包目录

```powershell
.\build\bin\HacPack.exe pack .\modules .\build\modules_test.hac
```

期望：

- 退出码为 `0`
- 输出包含 `version=V2` 和 `plugins=[NameList,CRC32]`
- 生成 `build/modules_test.hac`

### 3.2 列表与 HASH

```powershell
.\build\bin\HacPack.exe list .\build\modules_test.hac
.\build\bin\HacPack.exe list .\build\modules_test.hac --hash
```

期望：

- 普通列表含列：`id offset size name`
- `--hash` 列表含列：`id offset size hash(CRC32) name`
- `hash(CRC32)` 为 8 位十六进制字符串

### 3.3 比较包与目录

```powershell
.\build\bin\HacPack.exe compare .\build\modules_test.hac .\modules
```

期望：

- 完全一致时返回 `0`
- 输出 `MATCH\tpack and directory are identical`

### 3.4 解包并回比

```powershell
.\build\bin\HacPack.exe unpack .\build\modules_test.hac .\build\modules_unpacked
.\build\bin\HacPack.exe compare .\build\modules_test.hac .\build\modules_unpacked
```

期望：

- `unpack` 返回 `0`
- 二次 `compare` 返回 `0` 且输出 `MATCH`

---

## 4. 异常测试建议

### 4.1 非 V2 包

对一个非 V2 文件执行 `list` / `compare`：

```powershell
.\build\bin\HacPack.exe list .\build\not.hac
```

期望：

- 返回非 `0`
- 输出：`Unsupported pack version (only V2 is supported)` 或对应错误信息

### 4.2 缺少 NameList 插件映射

构造无 `NameList` 的测试包后执行：

```powershell
.\build\bin\HacPack.exe compare <pack> <dir>
```

期望：

- 返回非 `0`
- 输出 `Missing name mapping for file id: ...`

### 4.3 路径安全

若包内名称包含绝对路径或 `..`：

```powershell
.\build\bin\HacPack.exe unpack <pack> <out>
```

期望：

- 返回非 `0`
- 输出 `Unsafe entry path in pack: ...`

## 5. 自动化测试（CTests）

当前已接入用例：

```powershell
ctest --test-dir build --output-on-failure -C Debug
```

期望：

- `hacpack_pack_compare_source` 通过
- `hacpack_pack_unpack_compare_source` 通过
- 汇总 `100% tests passed`

---

## 6. 验收标准

满足以下条件即可认为 `main.cpp` 功能测试通过：

1. `pack/list/compare/unpack` 四个命令全部可执行。
2. `compare` 能在一致时返回 `0`，不一致时返回非 `0`（当前为 `2`）。
3. `list --hash` 能输出 `CRC32`。
4. CTest 两个集成用例全部通过。
